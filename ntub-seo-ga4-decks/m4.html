The Pipeline · 任何一關卡關、頁面就消失

爬取 → 索引 → 排名

三件事完全不一樣。誤把它們當同一件，後面的優化都會誤判。

Stage 01
爬取 Crawling
Googlebot 訪問頁面、讀取 HTML。能不能被爬到，由 robots.txt、Crawl Budget、伺服器回應決定。
Stage 02
索引 Indexing
把頁面加入 Google 資料庫。爬到不等於收錄；重複內容、品質太低、Noindex 都會被刷掉。
Stage 03
排名 Ranking
使用者搜尋時，根據演算法決定出現在第幾位。M2–M3 學的內容相關性都在這關打。
本週的工作集中在前兩關。沒被爬、沒被索引，內容寫得再好也不會出現在搜尋結果。
— GSC 的「索引涵蓋範圍」報表就是看這兩關

四個 WordPress 特有 的現場

Eater 01
標籤頁爆炸
100 篇 × 10 標籤 = 1,000 個低品質 /tag/xxx/。
修復：Yoast → 進階 → 標籤頁 noindex。
Eater 02
分類分頁
一直生 /category/xxx/page/2/，內容幾乎一樣。
修復：Canonical 指第一頁，或 noindex。
Eater 03
站內搜尋頁
每次搜尋產生 /?s=keyword。
修復：robots.txt 加 Disallow: /?s=。
Eater 04
作者存檔頁
單人部落格也會生 /author/admin/。
修復：Yoast → 進階 → 作者存檔 noindex。

The Three Tools · 控制爬蟲的三件套

Compare · 用途 / 語法 / 何時用

三件套 · 各有各的工作

Tool 01 · 路口柵欄
Robots.txt
用途：叫爬蟲不要進來某個資料夾。
語法：User-agent: * + Disallow: /wp-admin/
何時用：整個目錄都不想被爬（後台、管理頁）。
注意：Disallow 不保證不被收錄。
Tool 02 · 進門但別記
Noindex
用途：讓爬蟲爬，但不要收錄。
語法：頁面 head 內 <meta name="robots" content="noindex">
何時用：謝謝頁、登入頁、標籤頁、測試頁。
記住：要確保不進索引，這個比 robots 可靠。
Tool 03 · 邀請名單
Sitemap.xml
用途：主動告訴 Google 請看這些頁。
語法：XML 檔列出所有 URL，網址通常是 /sitemap_index.xml。
何時用：新站、新文章上線、結構大改後。
注意：Sitemap 是邀請、不是保證收錄。

The Trap · Robots ≠ Noindex

Disallow 不等於 Noindex

Robots.txt 的 Disallow 是「不要爬」。
Noindex 是「爬了但不要收錄」。
如果一個頁面被 Disallow，但別的網站連結到它—— Google 仍可能把那個 URL 放進索引（只是沒看內容）。

要確保某頁絕對不進搜尋結果，用 Noindex，不用 Disallow。
— Google 官方建議
Class Drill · 課堂練習

15 分鐘・三件事

  1. 瀏覽器打 http://ntub-demo.local/robots.txt，確認 /wp-admin/ 已被 Disallow。
  2. WP 後台 → 設定 → 閱讀，確認「阻擋搜尋引擎」沒有勾選。
  3. Yoast → 一般 → 功能，確認 XML Sitemap 已啟用，記下 sitemap 網址。
Verify·能說出 Robots.txt 與 Noindex 的差異

五步驟 把 GSC 接上線

這五步是固定的——之後每個學生上線真實站時都會走一次。

Step 01
登入 GSC
前往 search.google.com/search-console，用要管理這站的 Google 帳號登入。
Step 02
新增資源
點「新增資源」，選「網域」（推薦，含子網域與 HTTPS）或「URL 前置字元」。
Step 03
驗證所有權
最常用 HTML 標籤：把 GSC 給的 meta 貼到首頁 head；Yoast → 一般 → 網站認證即可。
Step 04
提交 Sitemap
索引 → Sitemap → 輸入 /sitemap_index.xml（Yoast 預設）→ 提交。
Step 05
等資料 24–72 小時
GSC 不是即時報表。隔天回來看「索引」與「成效」逐步開始有數字。

The Bridge · 把搜尋與站內行為串起來

GSC ＋ GA4
= 從曝光到轉換

GSC 看的是搜尋端：曝光、點擊、CTR、排名。
GA4 看的是站內行為：停留、頁面、轉換。
兩端串起來，才知道哪些關鍵字真的有商業價值。

路徑：GA4 → 管理 → 產品連結 → Search Console → 連結後 24–48 小時資料同步，只含 Organic Search。

三件套 · 三個欄位的工作說明書

Title Tag
長度：50–60 字元（中文約 25–30）
寫法：關鍵字放前面、每頁獨特、品牌用 | 放結尾
常見錯：所有頁面同一個 Title、塞 5 個關鍵字像垃圾郵件
≤ 60 chars
Meta Description
長度：120–155 字元（中文約 50–70）
寫法：含關鍵字（搜尋詞會加粗）+ 行動詞（立即查看 / 免費索取）
常見錯：跟頁面內容不符——Google 會自動換掉你寫的
≤ 155 chars
H1
數量：每頁只能一個
寫法：含主關鍵字、可與 Title 不完全相同（H1 文章語氣、Title 廣告語氣）
常見錯：把 Logo 或裝飾大字放進 H1
× 1 / page
H2 / H3 補充
原則：按順序使用（H1 → H2 → H3，不跳號）
用法：H2 涵蓋語意相關詞，幫 Google 抓 Featured Snippet
常見錯：整篇沒有 H2，長文一塊到底
structure

The Drill · 把上一節的 5 個關鍵字拿出來

20 分鐘 · 把 Yoast
從紅推到綠

  1. 打開 ntub-demo 後台 → 文章 → 編輯「Hello NTUB」。
  2. 依你選的演練網站定位，重寫 H1 標題。
  3. Yoast 區塊填 SEO 標題（≤ 60 字、關鍵字提前）。
  4. Yoast 區塊填 Meta Description（含關鍵字 + 行動詞）。
  5. 內文加 2–3 個 H2 結構化章節。
  6. Yoast → 設定 重點關鍵字（Focus Keyword），讓它分析。
