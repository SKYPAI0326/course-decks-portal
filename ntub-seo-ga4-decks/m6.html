Concept · Larry Page 的原始假設

每一個連結，
都是一張 選票

Google 把網路看成一張投票圖：A 連向 B，等於 A 把一份信任分配給 B。 被越多高品質頁面連結，PageRank 分數越高，排名越好。

Anchor Text（連結文字）也傳遞語意：
「台灣 SEO 專家」這個 Anchor，告訴 Google 目標頁面與「台灣 SEO」相關。

品質 > 數量

一個《紐約時報》的連結，
勝過一千個垃圾部落格連結。

Five Quality Signals · 連結品質判斷的五個維度

一個好連結，同時看五件事

01 · 來源權威性 高 DA / DR 的媒體、政府、學術網站，比個人小站重得多。 DA / DR 30+
02 · 主題相關性 同行業或相關內容的網站連結，比完全不相關的更有效。 topic match
03 · 連結位置 內文 Contextual 連結 > 頁尾 Footer / 側欄 Sidebar 連結。 in-content
04 · Anchor Text 自然 多樣化、描述性、不過度優化（不要全部都是「點這裡」也別全塞關鍵字）。 diverse
05 · do-follow 訊號 do-follow 才會傳遞 PageRank；no-follow 仍有品牌曝光，但權重偏弱。 rel attribute
反例提醒：不要追求「短期快速大量連結」。Penguin 演算法 就是為了打擊這種行為而設計的。

The Choice · 短期效果 vs 長期資產

贏得連結，還是 操弄連結？

黑帽 · 操弄連結

短期有效、長期高風險

  • 付費購買連結
  • 連結農場與 PBN 私人網絡
  • 大量低品質目錄登錄
  • 被 Penguin 抓到 → 排名大跌或手動懲罰
白帽 · 贏得連結（5 個方向）

慢、但是真資產

  • 創作 Linkable Assets（原創研究 / 工具 / 數據）
  • 媒體公關 → 報導與引用
  • 客座文章引用自己頁面
  • 資源頁面置換 / 修復斷鏈
  • 外展 5 步：找目標 → 確認資產 → 個人化開場 → Follow-up → 追蹤

Quality Rater Guidelines · 不是排名演算法，但間接影響排名

四個字母，四種信任

E · Experience
經驗
作者是否真的用過、做過、體驗過？第一手經歷的內容比轉述更可信。
E · Expertise
專業
作者是否具備該領域的專業背景、學歷、訓練？標明資歷有助於評估。
A · Authoritativeness
權威
這個品牌 / 作者在業界是否被信任、被引用？外部反向連結是關鍵訊號。
T · Trustworthiness
可信度
About 頁面 / 聯絡資訊 / 隱私權聲明 / HTTPS / 資料來源引用是否齊全？
YMYL 提醒：醫療、法律、金融類網站對 E-E-A-T 要求最嚴；一般電商與部落格門檻較低，但仍是長期該做的功課。

Architecture · 開始建之前先弄懂結構

帳戶 → 資源 → 串流

不搞清楚這三層，後面幫客戶設定 GA4 一定會搞混。

Layer 01 · Account
帳戶
最高層 · 組織單位 · 管理權限。一個 Google 帳號可以有多個帳戶。
Layer 02 · Property
資源
一個網站 / App = 一個資源 · 資料隔離。免費方案上限 2,000 個 / 帳戶。
Layer 03 · Stream
資料串流
實際資料來源 · 一個網站串流 + iOS / Android App 串流可並存。
實務舉例：ACME 品牌帳戶 → 官網資源 / App 資源 / 子品牌資源 → 各自的串流。
測試資料一定用獨立資源，不要污染生產資料。

13 / 20 講義 · m6-2-ga4-account
Enterprise Setup · 企業級帳戶都這樣命名

命名規則 + 權限分級

命名範式 [品牌]-[用途]-[環境]：例如 ACME-Website-PROD vs ACME-Website-STAGE。測試資料一律走 STAGE 資源。 naming
Admin · 管理員 能改所有設定（建串流、刪資源、改篩選器）。少數人才該有。 role
Editor · 編輯 能建事件 / 轉換 / 自訂報表。多數行銷團隊主力用這層。 role
Analyst · 分析 能建探索分析、可分享報表。不能改設定。 role
Viewer · 檢視 只能看報表 — 多數行銷人員 / 客戶 / 主管。 role
外包提醒：外部顧問通常給 Editor 而不給 Admin。
這條是合作信任的安全邊界 — 委任結束後，移除權限不會留下後門。

The Trade-off · 速度 vs 控制

Site Kit 還是 手動 gtag？

A · Site Kit by Google

快、官方、零代碼

  • WP 後台一鍵授權 + 連結
  • 不需碰 PHP / 主題檔
  • 順便連 GSC / PageSpeed
  • 本地 .local 站授權可能跳警告（按繼續即可）
  • 之後改走 GTM 記得移除，避免重複計算
B · 手動插入 gtag.js

慢、可控、走 head

  • 把 gtag 片段放進主題 functions.php
  • 或用「Insert Headers and Footers」外掛
  • 必須貼在 <head>，不要塞文章內容區
  • 把 G-XXXXXXXXXX 替換成自己的 ID
  • 未來轉 GTM 時不會被 Site Kit 污染
